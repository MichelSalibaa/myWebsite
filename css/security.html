Common risks

Web users and developers face many common risks: phishing and social engineering, cross‑site scripting (XSS), cross‑site request forgery (CSRF), insecure direct object references, broken authentication, and supply‑chain attacks. On the network layer, man‑in‑the‑middle attacks and unencrypted connections expose users’ data. Server misconfiguration and weak credentials remain frequent root causes of breaches.

Safe Practices for users

  • Prefer sites that use HTTPS (look for a padlock icon and valid certificate).
  • Use unique, strong passwords and a password manager; enable multi‑factor authentication where available.
  • Be cautious with email links and attachments; verify sender identity before taking sensitive actions.

Best Practices for Developers

  1. Adopt secure-by-design principles: validate inputs, use prepared statements/parameterized queries to prevent injection, and enforce least privilege on services.
  2. Use HTTPS everywhere with HSTS, set Content Security Policy (CSP), and implement secure cookie attributes (HttpOnly, Secure, SameSite).
  3. Keep dependencies up to date and monitor for supply‑chain vulnerabilities; use reproducible builds and signed packages where possible.
